# 管理员单页接入骨架

本目录提供微信小程序管理员/客服视图，用于默认单管理员快速查看所有桌、房、主局状态。当前页面默认可使用 `index.js` 内 mock 数据展示；当 `app.globalData.apiBaseUrl` 配置后，会通过 `frontend/services/api.js` 拉取真实后端数据。

## 页面能力

- 顶部汇总今晚桌/房数量、在线人数、爆照数量、需关注数量。
- 桌/房列表展示桌号、房间位置、人数、状态、局头、成员头像昵称、最近消息、消息数量、爆照数量。
- 支持按全部、主局中、待开局、需关注筛选。
- 支持切换桌子并在详情区查看成员列表、入局码、入局链接和最近动态。
- 支持复制入局码、复制入局链接、弹窗展示入局信息。
- 支持管理员查看、复制、联系用户微信。
- 在详情区展示对用户可见的管理员微信。
- 联系规则明确为管理员和用户可互加微信，不提供用户之间互加微信入口。

## 待接后端接口点

1. `GET /api/admin/tables?partyId=party_demo&adminId=admin_mimei`
   - 已完成基础接入。
   - 返回桌/房基本信息、状态、管理员、成员摘要、爆照计数、最近消息、入局码。

2. `GET /api/room?partyId=...&tableId=...`
   - 用于点击桌子后拉取更完整的详情。
   - 当前管理员页直接使用列表中的详情，后续可按需接这个接口获取更完整的成员、消息和备注。

3. `POST /api/contact/request`
   - 已完成后端权限规则：管理员可联系用户，用户可联系管理员，用户与用户互加微信返回 `403`。

4. `POST /api/messages`
   - 可用于后续在管理员侧补充客服消息或系统提示。

5. `POST /admin/tables/{id}/join-code`
   - 用于重新生成或刷新入局码。
   - 当前 UI 只做展示与复制，后续可在详情区增加刷新按钮。

6. `GET /admin/overview`
   - 用于顶部汇总统计。
   - 当前统计已由桌/房列表聚合，后续高并发时可改独立接口。

7. 实时更新通道
   - 用户房间页已接入 WebSocket；管理员页后续可复用同类通道。
   - 用户离开小程序后的通知栏提醒走微信订阅消息。
   - 用于同步成员进出、爆照数量、最近消息、桌房状态变化。

8. `GET /admin/profile`
   - 用于获取管理员昵称、管理员微信、是否对用户可见。
   - 当前后端通过主局接口返回默认管理员资料；用户端只能展示管理员微信，不展示其他用户微信。

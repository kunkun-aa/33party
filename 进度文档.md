# 33party 酒吧拼台小程序进度

更新时间：2026-05-31 15:44

## 当前状态

已完成一个可本地运行的 MVP 骨架，覆盖：

- 微信小程序入口配置：用户房间页、管理员页。
- 用户扫码/链接进入主局：解析 `scene`、`partyId`、`tableId`、`inviteCode`。
- 首次进入资料完善：头像、昵称、本地缓存，真实接口模式下写入后端。
- 用户局内体验：展示局头/管理员、管理员微信、酒吧地址、桌/房信息、成员、聊天室。
- 聊天能力：文字、语音、照片、消息点赞、照片闪照。
- 聊天 UI：点赞已收敛到消息头部的小文字，避免占据聊天区；照片默认普通发送，只有勾选“闪照”才按闪照发送。
- 聊天输入：文字、语音、照片已统一在同一个输入组件内，语音和照片作为输入框旁的快捷按钮。
- 聊天布局：当前用户自己发送的文字、语音、照片消息会靠右展示。
- 用户房间页视觉：已移除顶部重复的酒吧名/局头小字/右侧局头头像，以及聊天标题右侧的冗余“文字 / 语音 / 照片”说明。
- 用户房间页信息按钮：管理员微信“复制”和酒吧地址“地图”已从原生 `button` 改为 `view` 触控按钮，并与语音、照片、发送统一为 `64rpx` 圆形规格，避免微信默认按钮最小宽度把它们撑成长条。
- 用户聊天输入修正：语音/照片/发送按钮均改为非 `button` 组件，避免微信默认按钮样式把输入栏挤成两行、大椭圆或不规则形态；闪照选项改为轻量开关；发送按钮已与语音、照片按钮统一为 `64rpx` 圆形规格。
- 管理员/客服视图：桌房列表、占位人数、入局人数、未占位人数、成员、最近消息、照片数量、入局码/链接。
- 管理台视觉升级：页面改为 `#080B10` 极夜深色背景，卡片/详情面板使用 `#121620`，状态色按未满/已满/需关注区分，桌号放大为第一视觉锚点。
- 管理台交互升级：成员行只保留“联系并复制”和“管理”两个入口，占位和踢走移动到二级 Action Sheet，降低现场误触风险。
- 管理台微互动：刷新按钮增加旋转状态；复制微信、入局码、链接后在按钮原位显示“✓ 已复制”，减少遮挡视线的 toast。
- 管理台按钮尺寸优化：全局按钮、刷新、筛选、入局码/链接、成员操作按钮均下调视觉尺寸，避免暗色面板里按钮抢占信息层级。
- 管理台移动端布局：默认单列、成员行纵向、座位/链接/微信区域可换行，避免页面元素横向超出显示屏。
- 管理员资料编辑：管理台可直接修改对用户可见的管理员微信号，并写入后端 `admins.wechat_id`。
- 创建局能力：管理台可创建新局，填写局名称、桌/房号、人数、酒吧名称、酒吧地址；后端会创建酒吧、主局、默认桌台和入局码。
- 房间信息：按占位人数显示“人数未满”或“人数已满”，不再暴露数据库 `open` 状态。
- 地图能力：酒吧地址旁提供微信 `wx.openLocation` 打开地图。
- 背景音乐：台内默认尝试播放背景音乐，并提供播放/暂停控制。
- 用户资料：用户端显示性别，管理员端不展示性别。
- 占位管理：用户入局默认未占位，管理员可将成员设为占位，也可踢走不来的用户。
- 微信联系规则：用户只能添加管理员微信，管理员可以联系用户；用户与用户互加微信在后端返回 `403`。
- 后端与数据库：Python 标准库 HTTP 服务 + SQLite，自动建表和种子数据。

## 数据库表

已创建：

- `bars`：酒吧信息与地址。
- `admins`：默认管理员/局头资料与微信。
- `parties`：主局。
- `party_tables`：桌/房。
- `users`：小程序用户资料。
- `party_members`：用户入局/入桌关系，含 `ghost` 未占位和 `seated` 已占位状态。
- `messages`：文字、语音、照片、系统消息，含点赞数与闪照过期时间。
- `contact_requests`：微信联系请求记录。

## 微信联系权限

已落实到后端接口层：

- `user -> admin`：允许，返回管理员微信。
- `admin -> user`：允许，返回用户微信。
- `user -> user`：禁止，返回 `403` 与“平台不允许用户之间互加微信”。

前端呈现：

- 用户端只展示管理员微信和“复制添加”入口。
- 用户端成员列表不展示其他用户微信，也没有互加按钮。
- 管理员端可查看、复制、联系用户微信，并展示对用户可见的管理员微信。
